# プロジェクト構成と役割の整理

SaaS・配布用ツール化への移行に伴い、本システムは現在 **3つのGASプロジェクト** に分割・整理されています。
今後の開発やデバッグにおいて混乱を防ぐため、それぞれの役割と関係性をここに定義します。

---

## 1. ライセンス管理サーバー (`gas_license_manager`)

*   **役割:** 全利用者のライセンスキーとユーザー情報を一元管理する「Web API（データベース）」です。
*   **実体:** 開発者が管理する「AI仕訳生成ツールライセンス管理」スプレッドシートに紐付く独立したGASプロジェクト。
*   **特徴:** 
    *   Webアプリとしてデプロイされ、外部からの通信（`verify`, `activate` リクエスト）を受け付けます。
    *   各クライアントの「Googleドライブ ルートID（環境ID）」と「ライセンスキー」の紐付けを行い、不正なコピーや他環境での使い回しをブロックします。

## 2. 開発マスター 兼 共有ライブラリ本体 (`gas_proj`)

*   **役割:** AI解析や画面表示など、ツールの**すべてのコアロジックとUI（HTML）を持つ心臓部**です。
*   **実体:** 開発者が管理する「マスター用スプレッドシート」に紐付くGASプロジェクト。
*   **特徴:**
    *   **今後の開発場所:** 機能追加やバグ修正など、すべてのプログラミング作業はこのプロジェクト内で行います。
    *   **ライブラリ化:** 開発が完了した時点で「新しいバージョン」としてデプロイ（発行）されます。
    *   これにより、ソースコードの隠蔽（コードプロテクト）と、全ユーザーへの一斉アップデート機能を実現します。

## 3. 配布用クライアント (`gas_client`)

*   **役割:** お客様（ユーザー）に実際に使っていただく「配布用スプレッドシート」の裏側で動く、**極めて薄い連携プログラム（殻）**です。
*   **実体:** ユーザーがコピーして使う「配布用スプレッドシート（設定空・データ空）」に紐付くGASプロジェクト。
*   **特徴:**
    *   独自の処理は一切持ちません。メニューなどの呼び出し口を持つのみで、処理はすべて `2. gas_proj`（JournalLib）へ丸投げします。
    *   `google.script.run` などの画面通信を中継するための「スタブ（空の関数）」を持っています。
    *   **デバッグ時の注意:** 現状、このクライアントとライブラリ（gas_proj）間のHTML通信における「実行コンテキストの違い」によってエラー（not a function 等）が発生しており、これに対処する必要があります。
